はじめに#

hadv とは何か#

hadvとは、Hugo, Asciidoctor, Docker, VSCodeを上手に使って構築したドキュメント執筆環境を私が勝手に名付けたものです。 ちなみに、タイトルのhadv-bookとは「hadv環境上にHugoのbookテーマを使って構築したドキュメント執筆環境のひな形」を指します。

インターネットで検索するとHAdV（Molecular Evolution of Human Adenovirus）と記述される「ヒトアデノウィルス」なんてものが出てきますが、 ジャンルが違うので問題ないかと。

hadv の特徴#

  • VSCodeのdevcontainer機能を使用して、ローカル環境を汚さずに快適なドキュメント執筆環境が手に入ります。

  • Hugo, Asciidoctorそれぞれの公式Dockerイメージを拡張して使用しています。

  • devcontainerの設定を変更するとベースとなる公式Dockerイメージのバージョンを更新できます。
    （更新した場合の動作までは保証できませんので、ご了承ください）

  • Hugo（静的サイトジェネレーター）を使ってドキュメントを静的なサイトとして公開できます。

  • Hugoで公開するドキュメントをAsciiDoc形式（Asciidoctorを使用）で記述することができます。

  • VSCodeでMarkdown形式、AsciiDoc形式のドキュメントをプレビューしながら記述できます。

このドキュメントについて#

このドキュメントでは、hadv-book 構築に必要だった手順をすべて公開します。
無駄に長くなってしまいそうなので説明はあえて少なくします。

hadv-bookではHugoのbookテーマを使用させてもらいますが、別のテーマを使ってhadv環境を作ることもモチロン可能ですので、気が向いたらチャレンジしてみてください。

1-3. 各ファイルの内容#

[1] hadv-asciidoctor/Dockerfile#

ARG DOCKER_ASCIIDOCTOR_VERSION=latest (1)
FROM asciidoctor/docker-asciidoctor:${DOCKER_ASCIIDOCTOR_VERSION} (2)

LABEL version="20210324"
LABEL description="Asciidoctorの公式Dockerイメージをカスタマイズ。"

RUN apk --no-cache add docker-cli git-subtree \ (3)
  && gem install --no-document asciidoctor-html5s asciidoctor-rouge (4)
RUN ln -s /documents /src (5)
COPY hugo /usr/local/bin
1 デフォルトのバージョン
2 [6] docker-compose.yml 内にDOCKER_ASCIIDOCTOR_VERSIONの指定がある。
3 『Docker from Docker』のためにdocker-cliを追加。任意でgitの機能を追加。
4 Asciidoctorの拡張機能を追加（これ以外のHugoで使用可能とされる拡張機能はエラーが発生していまうため追加せず）。
5 hadv-hugoからhadv-asciidoctorのコンテナを使用する時のパスの違いを吸収するためにシンボリックリンクを設置。
